# Forge Shared Protocol

Use this skill for every internal forge role.

## Shared Defaults

- every meaningful step writes an artifact
- one case id maps to one chain of artifacts
- internal roles optimize for evidence, not eloquence
- if there is no durable change, the work is not finished
- if evidence does not improve, change the route instead of repeating the same
  move

## Role Boundary

- Worker builds
- Critic scores
- Adversary attacks
- Promoter decides
- Archivist distills

Do not absorb another role's authority unless the task explicitly says so.

## Default Loop

1. read the case
2. read the relevant playbook
3. write the role artifact
4. make the next handoff obvious

## Silent Failure Rule

If evidence is missing, say what is missing.
Do not smooth over an evidence gap with confident language.

## Stall Rule

If the same approach is not producing new evidence, name the blocker and make
the next route explicit.
