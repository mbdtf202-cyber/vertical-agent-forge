OpenClaw 生产级工具包

把垂直 agent 做成产品，而不是大 prompt。

Vertical Agent Forge 为 OpenClaw 增加一个 durable improvement loop： 多角色 agent、release gate、artifact 合同，以及热插拔安装方式。

角色清晰

把用户交付和质量控制拆开，不让一个 agent 同时承担两种责任。

  • app-main 面向用户
  • app-forge 编排改进
  • 子 agent 保持有边界、可追责

持久化编排

直接复用 OpenClaw 现有 runtime：task、 continuous-worker、wake、subagent。

  • 进度跨多轮持久存在
  • wake 不依赖单次聊天
  • 核心证据留在文件里

发布纪律

每个改动都要经过 proposal、critique、adversarial review 和 promotion。

  • candidate artifact
  • scorecard 与 attack report
  • decision 与 distillation record

对比

为什么它是产品，而不是 prompt 技巧

单 Prompt Agent

  • 一个 runtime 包办所有事
  • 质量规则容易漂移
  • 回归难以审计

临时多 Agent

  • 有分工，但 release discipline 弱
  • artifact 不稳定
  • 晋升逻辑常常是隐式的

Vertical Agent Forge

  • durable task 与 wake runtime
  • 有边界角色与明确门禁
  • 可复用 artifact、memory 和 release assets

Examples

一键初始化的领域模板

SaaS Support

账单策略、升级降级、退款和 plan eligibility。

node ./bin/vertical-agent-forge.mjs init --domain saas-support

Travel Concierge

shortlist-first 旅行推荐与路线 tradeoff。

node ./bin/vertical-agent-forge.mjs init --domain travel-concierge

Research Ops

证据整理、研究分诊与内部分析流程。

node ./bin/vertical-agent-forge.mjs init --domain research-ops

CLI 生命周期

安装、激活、检查、卸载

node ./bin/vertical-agent-forge.mjs install
node ./bin/vertical-agent-forge.mjs activate
node ./bin/vertical-agent-forge.mjs init --domain saas-support
node ./bin/vertical-agent-forge.mjs upgrade
node ./bin/vertical-agent-forge.mjs doctor
node ./bin/vertical-agent-forge.mjs uninstall
