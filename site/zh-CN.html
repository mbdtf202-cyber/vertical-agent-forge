工厂角色

把交付、编译、评测、发布和监控拆开，而不是把一切都藏进一个 prompt。

  • app-main 面向用户
  • app-domain-compiler 负责编译领域包
  • app-evaluator、app-deployer、app-observer 负责闭环

编译后的领域系统

原始业务资料会被编译成 policies、glossary、action catalog、eval assets、routing 和 metrics。

  • 业务事实留在文件里，不只留在聊天里
  • 编译产物可测试、可回归
  • 内置 SaaS Support 参考包

控制面自动化

每个改动都要经过 validation、rollout evidence、connector checks、promotion gate 和 rollback。

  • snapshot、jobs、metrics、connectors、incidents
  • 默认严格激活
  • canary-first，上线异常自动回滚

对比

为什么它是产品，而不是 prompt 技巧

单 Prompt Agent

  • 一个 runtime 包办所有事
  • 质量规则容易漂移
  • 回归难以审计

临时多 Agent

  • 有分工，但 release discipline 弱
  • artifact 不稳定
  • 晋升逻辑常常是隐式的

Vertical Agent Forge

  • 编译后的 domain pack + runtime control plane
  • 有边界角色、明确门禁、显式 anti-stall 切路
  • 可复用 artifact、memory、release assets 和 rollback 逻辑

Examples

一键初始化的领域模板

SaaS Support

账单策略、退款规则、plan eligibility、ticket fixtures 和 metric thresholds。

node ./bin/vertical-agent-forge.mjs bootstrap --domain saas-support

Travel Concierge

shortlist-first 旅行推荐与路线 tradeoff。

node ./bin/vertical-agent-forge.mjs init --domain travel-concierge

Research Ops

证据整理、研究分诊与内部分析流程。

node ./bin/vertical-agent-forge.mjs init --domain research-ops

CLI 生命周期

Bootstrap、validate、evaluate、ship、rollback

node ./bin/vertical-agent-forge.mjs bootstrap --domain saas-support
node ./bin/vertical-agent-forge.mjs doctor --deep
node ./bin/vertical-agent-forge.mjs validate
node ./bin/vertical-agent-forge.mjs run-evals --case CASE-20260317-001 --stage canary
node ./bin/vertical-agent-forge.mjs deploy --case CASE-20260317-001 --stage live
node ./bin/vertical-agent-forge.mjs rollback --case CASE-20260317-001 --reason "canary breach"
node ./bin/vertical-agent-forge.mjs status --deep
